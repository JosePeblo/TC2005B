Preguntas de JavaScript

¿Por qué es una buena práctica usar JavaScript para checar que sean válidos los inputs de las formas antes de enviar los datos al servidor?

Para darle una buena experiencia a los usuarios e informarlos de las restricciones que tienen los datos que pueden entrar en nuestro servidor. Aun que vayamos a revisar los datos dentro del servidor es importante que los usuarios con intrerés legítimo puedan interactuar con la interfaz.

¿Cómo puedes saltarte la seguridad de validaciones hechas con JavaScript?

Cualquier navegador puede bloquear JavaScript, pero usualmente puede hacer que las páginas dejen de funcionar en especial si fueron creadas con un framework que genere el contenido dinámicamente.

Si te puedes saltar la seguridad de las validaciones de JavaScript, entonces ¿por qué la primera pregunta dice que es una buena práctica?

Realmente, se pueden realizar las peticiones de http desde software externo al navegador, sin embargo los es importante que en el servidor haya validaciones para evitar que se abuse de este.
Como ya lo mencionamos, tenemos que verificar que el usuario que no intente atacarnos pueda interactuar correctamente
