¿Cuáles son las partes de un URL? Ref*

  • Un url se compone de un protocolo como: http://, https://, mqtt://

  • Un sub-dominio como: www

  • Un dominio, que es el nombre de la página o la dirección ip de esta como: google.com

  • Una ruta la cual se separa por / como si se tratara de directorios como: /cursos/progra/etc

  • Parametros los cuales son una serie de llaves y valores la cual se comienza con un ? y después se nombra la llave, un signo de = y el valor. Se coloca un & para colocar un parámetro nuevo ?nombre=Juan&apellido=Monroy

  • La etiqueta que se denota por un # y después se nombra el id de un elemento al que se quiere hacer scroll instantáneamente #references

¿Cuál es el propósito de los métodos HTTP: GET, HEAD, POST, PUT, PATCH, DELETE? Ref*

El protocolo http nos provee de estos métodos para la transmisión de información a través de la web

  • Get se utiliza para recuperar datos
  • Head es como Get pero omitiendo el cuerpo de la respuesta
  • Post se utiliza para enviar informacion al servidor
  • Put envia informacion y la reemplaza, a diferencia de post, usar este metodo no crea informacion adicional en una petición
  • Patch es como Put pero este método puede reemplazar partes parciales de la informacion que quiere cambiar
  • Delete se utiliza para borrar un registro específico

No, ya que es probable que si se encontró con un error 404 (not found) es porque intentó entrar a un recurso que no existe y al que no se debería poder llegar desde la página o está intentando acceder a un recurso prohibido el cual por diseño responde con que no existe por cuestiones de seguridad.

Sí, ya que los errores del rango de los 500 corresponden a un error de servidor el cual tiene que ver con el como se programó este.

Preguntas Laboratorio 3

Como ingeniero de software ¿cuál es tu recomendación sobre el uso de !important en un CSS?

Se tiene que usar con cuidado ya es esta le dará prioridad a los estilos que le definas a ese elemento, lo cual, en un proyecto grande puede interferir con estilos de otros elementos.

Si se pone una imagen de fondo en una página HTML, ¿por qué debe escogerse con cuidado?

Para cuidar que el texto sea legible y no se confunda con alguna parte de la imagen.

Como ingeniero de software, ¿cuál es tu recomendación al elegir las unidades de un propiedad de estilo entre %, px y pt?

Diferenciar entre las unidades como px que nos permiten definir tamaños y espacios utilizando la cantidad de pixeles que tenga la pantalla del usuario. Unidades como % nos permiten hacer un cálculo basándonos en los elementos padres. La mayor recomendación usar todos los recursos que necesitemos para lograr un diseño responsivo que funcione en la mayoría de dispositivos.

¿Por qué el uso de una versión minimizada del CSS mejora el rendimiento del sitio?

Porque los espacios y saltos de línea son necesarios para que el programador pueda entender el código, sin embargo estos también consumen bytes y por ende también tienen que ser enviados al momento de que se entra a la página.

Minimisar nuestras hojas de estilos nos permite reducir la cantidad de informacion que se tiene que enviar y de esta forma hace más rápido el cargado de nuestra página.
